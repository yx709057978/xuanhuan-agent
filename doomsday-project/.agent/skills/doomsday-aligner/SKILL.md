# 末世重生漫剧一致性对齐检查 Skill

[技能说明]
    专业的末世重生漫剧一致性检查技能包，以大纲为核心基准，对已创作内容进行全面质量检查。覆盖10个检查维度，返回PASS/FAIL状态和具体问题清单，确保作品质量。

[检查时机]
    - 每批5集创作完成后自动调用
    - 用户明确要求检查一致性时
    - 检测到设定调整语时（如"推翻/改天灾/改金手指"等）
    - 检测到约束确立语时（如"必须/不能/要求/统一"等）

[核心能力]
    - **基准对齐**：以 outline.md 为核心基准，所有检查以大纲为标准
    - **多维度检查**：覆盖10个检查维度，全面评估内容质量
    - **问题定位**：精准定位到具体集数和具体问题
    - **状态返回**：返回PASS/FAIL状态，便于自动化流程控制
    - **修改建议**：提供具体的修改建议，便于后续修订

[检查维度]

    ## 维度1：剧情进度一致性
    **检查标准**：
    - 章节正文是否严格按照 chapter_index.md 的分集大纲展开
    - 每集的核心情节是否与目录简介一致
    - 是否遗漏了目录中规划的关键情节
    - 是否添加了目录中没有的情节（需评估是否合理）
    
    **检查方法**：
    1. 读取 chapter_index.md 中对应集数的简介
    2. 对比章节正文的实际内容
    3. 检查是否完整覆盖了简介中的要点
    
    **FAIL条件**：
    - 正文与目录简介严重不符
    - 遗漏目录中的关键情节
    - 添加了与主线无关的大量内容

    ## 维度2：天灾进程一致性
    **检查标准**：
    - 天灾阶段进展是否符合 outline.md 中的天灾进程体系规划
    - 主角在每集所处的天灾阶段是否正确
    - 天灾爆发时机是否符合大纲设定
    - 天灾名称、阶段是否前后一致
    - 异能等级（如有）是否符合规划
    
    **检查方法**：
    1. 读取 outline.md 中的天灾进程体系表格
    2. 检查每集中天灾阶段的描述
    3. 对比是否符合规划
    
    **FAIL条件**：
    - 天灾阶段与大纲规划不符
    - 天灾名称前后不一致
    - 异能等级进展与大纲差异过大

    ## 维度3：爽点分布一致性
    **检查标准**：
    - 是否符合 outline.md 中的爽点分布规划
    - 第10/20/30/40/50/60集是否有大爽点
    - 每5集是否有中爽点
    - 每2集是否有小爽点
    - 末世特色爽点（囤货展示、物资碾压、前世仇人打脸）是否到位
    
    **检查方法**：
    1. 读取 outline.md 中的爽点分布规划
    2. 检查对应集数是否有规划的爽点
    3. 评估爽点密度是否达标
    
    **FAIL条件**：
    - 关键集数（10/20/40/60）缺少大爽点
    - 连续5集没有中爽点
    - 连续3集没有小爽点

    ## 维度4：人物行为一致性
    **检查标准**：
    - 人物性格是否符合 character.md 的设定
    - 人物能力是否符合设定（不能突然会/不会某技能）
    - 人物关系是否符合设定
    - 人物出场时机是否符合设定
    - 前世仇人的行为是否符合设定（该恶的恶，该蠢的蠢）
    
    **检查方法**：
    1. 读取 character.md 中的人物设定
    2. 检查章节中人物的行为、对话、能力展现
    3. 对比是否符合人物小传
    
    **FAIL条件**：
    - 人物性格崩坏（与设定严重不符）
    - 人物能力突然变化（无合理解释）
    - 人物关系与设定矛盾

    ## 维度5：时间线一致性
    **检查标准**：
    - 事件发生的时间顺序是否合理
    - 末日倒计时是否正确（如"距离末日还有X天"）
    - 囤货时间是否合理
    - 不同事件的时间线是否冲突
    
    **检查方法**：
    1. 提取章节中的时间标记
    2. 检查时间线是否连贯
    3. 检查是否有时间冲突
    
    **FAIL条件**：
    - 时间线明显矛盾
    - 同一时间发生冲突事件
    - 末日倒计时错误

    ## 维度6：伏笔处理一致性
    **检查标准**：
    - 大纲中埋下的伏笔是否在正确时机揭示
    - 已埋伏笔是否被遗忘
    - 伏笔揭示是否与埋设时一致
    - 前世记忆闪回是否合理使用
    
    **检查方法**：
    1. 从 outline.md 中提取伏笔设定
    2. 检查章节中伏笔的处理
    3. 评估是否合理
    
    **FAIL条件**：
    - 伏笔被遗忘
    - 伏笔揭示与埋设矛盾
    - 伏笔揭示时机严重不当

    ## 维度7：金手指规则一致性
    **检查标准**：
    - 金手指规则是否与 outline.md 定义一致
    - 空间/系统使用是否符合已建立的规则
    - 空间大小、功能不能突然变化（无合理解释）
    - 金手指的限制条件是否被遵守
    
    **检查方法**：
    1. 从 outline.md 中提取金手指设定
    2. 检查章节中金手指的使用
    3. 对比是否符合规则
    
    **FAIL条件**：
    - 金手指规则前后不一致
    - 空间功能突然变化（无解释）
    - 违反已建立的限制条件

    ## 维度8：格式规范一致性
    **检查标准**：
    - 是否使用了正确的视觉描述符号（※△【】等）
    - 是否使用了末世特色符号（【空间面板】【闪回】【对比】等）
    - 每集是否以【卡黑】结尾
    - 字数是否在500-800字范围内
    - 场景数量是否在1-3个范围内
    
    **检查方法**：
    1. 检查视觉描述符号使用
    2. 检查结尾是否有【卡黑】
    3. 统计字数和场景数量
    
    **FAIL条件**：
    - 缺少必要的视觉描述符号
    - 结尾没有【卡黑】悬念
    - 字数严重超出/不足范围

    ## 维度9：写作风格一致性
    **检查标准**：
    - 是否符合末世重生漫剧的视觉化、快节奏风格
    - 对话是否简短有力（不超过20字）
    - 是否有拖沓的铺垫或废话
    - 是否有过多的心理描写
    - 前世今生对比是否强烈
    - 物资数值是否可视化展示
    
    **检查方法**：
    1. 检查对话长度
    2. 检查是否有冗余描写
    3. 评估节奏是否够快
    
    **FAIL条件**：
    - 对话过长（超过20字）
    - 存在大量铺垫或废话
    - 心理描写过多
    - 节奏拖沓

    ## 维度10：创作禁忌检查
    **检查标准**：
    - 是否有连续3集没有打脸/囤货/物资展示
    - 是否有连续5集没有天灾推进
    - 是否有任何集结尾没有悬念
    - 是否有过度文言文或中二台词
    - 是否有配角戏份过多
    - 是否有圣母行为（无底线原谅前世仇人）
    
    **检查方法**：
    1. 统计打脸/囤货频率
    2. 检查每集结尾
    3. 检查台词风格
    4. 评估配角戏份占比
    5. 检查主角是否恩怨分明
    
    **FAIL条件**：
    - 违反任何创作禁忌
    - 连续多集无爽点
    - 配角喧宾夺主
    - 主角圣母行为

[执行流程]

    第一步：读取基准文档
        1. 读取 outline.md（核心基准）
        2. 读取 character.md（人物设定基准）
        3. 读取 chapter_index.md（剧情进度基准）
        4. 读取待检查的章节正文

    第二步：逐维度检查
        按照10个维度依次检查：
        1. 剧情进度一致性
        2. 天灾进程一致性
        3. 爽点分布一致性
        4. 人物行为一致性
        5. 时间线一致性
        6. 伏笔处理一致性
        7. 金手指规则一致性
        8. 格式规范一致性
        9. 写作风格一致性
        10. 创作禁忌检查

    第三步：生成检查报告
        输出格式：
        ```
        ## 一致性检查报告
        
        **检查范围**：第X-X集
        **检查时间**：YYYY-MM-DD HH:MM
        **总体状态**：PASS / FAIL
        
        ### 检查详情
        
        | 维度 | 状态 | 问题描述 |
        |------|------|----------|
        | 剧情进度 | ✅/❌ | 具体问题（如有） |
        | 天灾进程 | ✅/❌ | 具体问题（如有） |
        | 爽点分布 | ✅/❌ | 具体问题（如有） |
        | 人物行为 | ✅/❌ | 具体问题（如有） |
        | 时间线 | ✅/❌ | 具体问题（如有） |
        | 伏笔处理 | ✅/❌ | 具体问题（如有） |
        | 金手指规则 | ✅/❌ | 具体问题（如有） |
        | 格式规范 | ✅/❌ | 具体问题（如有） |
        | 写作风格 | ✅/❌ | 具体问题（如有） |
        | 创作禁忌 | ✅/❌ | 具体问题（如有） |
        
        ### 问题清单（如有FAIL）
        
        **第X集**：
        - [维度名称] 具体问题描述
        - [维度名称] 具体问题描述
        
        **第Y集**：
        - [维度名称] 具体问题描述
        
        ### 修改建议
        
        1. 第X集需要修改：具体修改建议
        2. 第Y集需要修改：具体修改建议
        
        ---
        **结论**：PASS（通过检查，可写入文档）/ FAIL（需要修改后重新检查）
        ```

    第四步：返回检查结果
        - 如果所有维度都通过 → 返回 PASS
        - 如果任何维度失败 → 返回 FAIL + 问题清单 + 修改建议

[检查原则]

    - **大纲为核心**：所有检查以 outline.md 为最高基准
    - **严格但合理**：严格检查但不过度苛刻
    - **问题定位精准**：精确到具体集数和具体问题
    - **建议可操作**：修改建议要具体可执行
    - **效率优先**：快速完成检查，不影响创作流程

[与doomsday-skill的协作]

    检查-修改-再检查闭环：
    1. doomsday-skill 创作内容
    2. doomsday-aligner 检查一致性
    3. 如果 FAIL → doomsday-skill 根据问题清单修改
    4. doomsday-aligner 重新检查
    5. 直到 PASS → 写入文档

[注意事项]

    - 检查时必须读取所有相关基准文档
    - 不能只看单集，要结合上下文
    - 问题描述要具体，不能模糊
    - 修改建议要可操作，不能空泛
    - PASS/FAIL 状态要明确，不能模棱两可
    - 检查报告格式必须规范，便于后续处理
    - 末世重生特色：注意天灾进程、囤货逻辑、前世今生对比
    - 始终使用中文输出
