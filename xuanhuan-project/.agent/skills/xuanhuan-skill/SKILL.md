# 玄幻漫剧创作 Skill

> 专业的玄幻漫剧创作技能包，覆盖故事构思、人物塑造、章节规划、正文写作、内容修订全流程。

## When to Use This Skill

- 用户要求创作玄幻漫剧大纲时
- 用户执行 `/character`、`/catalog`、`/write` 指令时
- 用户提出修改意见或 xuanhuan-aligner 返回 FAIL 时
- 需要玄幻漫剧创作方法论和风格指导时

## 执行流程

### Step 1：识别创作阶段

| 触发条件 | 创作阶段 |
|---------|---------|
| 讨论故事大纲 / outline.md 不存在 | 大纲创作 |
| 执行 `/character` | 人物创作 |
| 执行 `/catalog` | 目录创作 |
| 执行 `/write [批次]` | 章节创作 |
| 用户提修改意见 / aligner 返回 FAIL | 内容修订 |

### Step 2：读取创作资源

根据阶段读取对应资源：

| 阶段 | 必读资源 | 上下文 |
|------|---------|--------|
| **大纲创作** | outline-method.md + output-style.md + templates/outline-template.md + examples/outline-example.md | 用户提供的创意信息 |
| **人物创作** | outline-method.md（人物塑造部分）+ output-style.md + templates/character-template.md + examples/character-example.md | outline.md |
| **目录创作** | outline-method.md + output-style.md + templates/chapter-index-template.md + examples/catalog-example.md | outline.md + character.md |
| **章节创作** | outline-method.md + output-style.md + templates/chapter-template.md + examples/chapter-example.md | outline.md + character.md + chapter_index.md + 已有章节 |
| **内容修订** | 对应阶段的方法论 + 模板 | 被修改文档 + 修改意见/FAIL 反馈 |

### Step 3：执行创作

遵循以下核心原则：

1. **模板遵循**：严格按 templates/ 中的格式结构
2. **风格统一**：严格遵循 output-style.md 的视觉化快节奏风格
3. **方法论指导**：严格遵循 outline-method.md 的创作法则
4. **示例参考**：参考 examples/ 的风格和质量标准

### Step 4：返回成果

- 大纲创作 → 写入 outline.md
- 人物创作 → 写入 character.md
- 目录创作 → 写入 chapter_index.md
- 章节创作 → 写入 chapters/Episode-XX.md
- 内容修订 → 更新对应文档

## 核心创作原则

### 漫剧特性（必须遵守）

| 规格 | 要求 |
|------|------|
| 总集数 | 60 集 |
| 每集时长 | 1-2 分钟 |
| 每集字数 | 500-800 字 |
| 结构 | 三幕式（1-20 / 21-45 / 46-60） |
| 单集节奏 | 起承转钩四段式 |

### 爽点密度（必须遵守）

```
每 2 集 = 1 个小爽点（嘲讽→反击/小喽啰被秒）
每 5 集 = 1 个中爆点（境界突破/打脸主要角色）
每 10 集 = 1 个大爆点（地位跃升/身份曝光）
第 20/40/60 集 = 超级爆点
```

### 写作风格（必须遵守）

- **视觉化优先**：一切为画面服务，每句话都能转化为视觉内容
- **节奏极快**：瞬间进冲突，无废话无铺垫
- **爽点密集**：每集必有打脸/升级/碾压
- **悬念强制**：每集结尾必须【卡黑】

### 视觉符号（章节创作必须使用）

| 符号 | 用途 | 示例 |
|------|------|------|
| `※` | 场景/环境 | ※ 青云宗大殿 |
| `△` | 动作/变化 | △ 叶凡一拳轰出 |
| `【特效】` | 视觉特效 | 【特效】金光冲天 |
| `【系统面板】` | 数值展示 | 【系统面板】境界：炼气→筑基 |
| `【卡黑】` | 悬念结尾 | 【卡黑】 |

## 质量标准

### 大纲质量

- [ ] 60 集三幕式结构完整
- [ ] 核心梗概一句话概括（身份+困境+金手指）
- [ ] 导语 150-200 字（废材开局+羞辱+觉醒+悬念）
- [ ] 爽点分布规划清晰（10/20/30/40/50/60 集）
- [ ] 境界体系清晰可视化

### 人物质量

- [ ] 主角四要素齐全（境界/身份+处境/背景+性格标签+外貌特征）
- [ ] 配角工具人化，推动情节
- [ ] 炮灰反派按出场集数分批

### 章节质量

- [ ] 500-800 字
- [ ] 使用视觉描述符号（※△【】）
- [ ] 起承转钩四段式结构
- [ ] 每集必有悬念结尾【卡黑】
- [ ] 对话不超过 20 字/句

---

## 创作禁忌

### ❌ 绝对禁止

- 前 5 集没有金手指
- 连续 5 集没有打脸
- 任何集结尾没有悬念
- 主角被真正压制超过 3 集
- 配角戏份过多，喧宾夺主
- 过度文言文（"吾乃天命之子"）
- 啰嗦描写、无意义形容词堆砌

### ✅ 必须做到

- 每集必有推进（升级/打脸/获宝/战斗）
- 打脸要狠，碾压要爽
- 金手指简单易懂
- 境界数值可视化
- 第 60 集必留续集悬念

## 快速参考

### 创作公式

```
玄幻漫剧 = 废材开局 + 金手指觉醒 + 快速升级 + 密集打脸 + 爽点爆发
```

### 单集节奏

```
起（开场冲突）→ 承（推进发展）→ 转（反转高潮）→ 钩（悬念结尾）
```

|------|------|------|

| 第 20 集 | 第一大高潮 | 超级爽点 |
| 第 40 集 | 第二大高潮 | 身份曝光 |
| 第 60 集 | 终极高潮 | 续集钩子 |

十集之内给金手指，二十集处大爆发

## 注意事项

1. **outline-method.md 是核心创作法则**，必须严格遵守
2. **output-style.md 的视觉化快节奏是强约束**，必须严格遵守
3. **templates/ 中的格式是必须遵循的**，任何偏离都可能导致问题
4. **examples/ 仅作为案例示意**，不要因此限制剧情创作能力
5. **始终使用中文创作**
6. **章节创作必须使用视觉描述符号**（※△【】等）
